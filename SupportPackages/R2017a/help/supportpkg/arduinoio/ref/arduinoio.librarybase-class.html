arduinoio.LibraryBase class

Package: arduinoio

Abstract class for deriving Arduino add-on libraries

expand all in page

Description

classdef MyAddonClass < arduinoio.LibraryBase makes MyAddonClass a subclass of the arduinoio.LibraryBase class.

An object with arduinoio.LibraryBase as a parent class is an Arduino® Add-On. The arduinoio.LibraryBase class provides your Arduino Add-On subclass several protected methods to interact with the arduino object, interact with hardware resources, and communicate with C++ code on the Arduino hardware board.

Properties

expand all

LibraryName — Identifier of the library
string

A unique identifier for an add-on library. The library name string uses the name of the package folder where the library files are located, followed by the name of the library, with a forward slash separating the two strings.

Example: 'ExampleLCD/LCD'

Data Types: char

DependentLibraries — List of dependent Arduino Add-On libraries
cell array of strings

A list of other Arduino Add-On libraries required by your Add-On library. The library name string uses the same syntax as the LibraryName property.

Example: {'ExampleLCD/LCD','ExampleAddon/HelloWorld'}

Data Types: cell

ArduinoLibraryHeaderFiles — List of dependent Arduino C++ library header files
cell array of strings

A list of all C++ header files required by your Add-On library.

Example: {'ExampleLCD/LCD','ExampleAddon/HelloWorld'}

Data Types: cell

CppHeaderFile — Identifier of the add-on C++ header file
string

The name of the C++ header file that contains your add-on class code.

Example: 'MyAddon.h'

Data Types: char

CppClassName — Identifier of C++ add-on class name
string

The name of the class name in the C++ header file.

Example: 'MyAddon'

Data Types: char
